# Agent Hub - 目标与愿景

## 愿景

**让每个人都能拥有真正懂自己的智能体。**

不是冷冰冰的工具，而是有性格、有记忆、能操作你数据的智能伙伴。

---

## 核心理念

| 理念 | 说明 |
|------|------|
| **Agent First** | 智能体优先，不是工具优先 |
| **完整人格** | 有灵魂的 AI，不是冷冰冰的机器人 |
| **数据主权** | 用户数据在本地，智能体只是操作者 |
| **开放生态** | 任何人都可以创建和分享智能体 |
| **规范通用** | 不绑定单一平台，多平台兼容 |

---

## 要解决的问题

### 1. 分发

用户如何发现、安装、更新智能体？

### 2. 规范

智能体包的标准格式是什么？如何做到通用？

### 3. 依赖

智能体依赖的 CLI 如何自动安装？

### 4. 发布

作者如何创建、测试、发布智能体？

## 规范设计思路

### 原则

1. **通用优先** - 不绑定 OpenClaw，其他平台也能用
2. **人类友好** - Markdown 为主，打开就能看懂
3. **程序可解析** - YAML 元信息，方便自动化
4. **渐进增强** - 核心简单，扩展可选

### 结构（草案）

```
my-agent/
├── agent.yaml           # 元信息（必需）
├── IDENTITY.md          # 身份（必需）
├── SOUL.md              # 性格（必需）
├── TOOLS.md             # 能力（必需）
├── MEMORY.md.template   # 记忆模板（必需）
├── USER.md              # 用户画像模板（可选）
├── WORKFLOW.md          # 工作流（可选）
├── skills/              # 技能包（可选）
├── avatar.png           # 头像（可选）
└── README.md            # 介绍（推荐）
```

### 元信息（agent.yaml）

```yaml
spec: agent/v1
name: 仓咚咚
id: cang
version: 1.0.0
author: gudong
description: 帮你管钱的小仓鼠
emoji: 🐹

cli:
  package: cang-cli
  version: ">=0.1.0"

tags:
  - finance
  - personal

runtime:
  - openclaw
```
