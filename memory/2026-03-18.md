## 咚咚家族项目发布

### dong-core 0.2.3 发布

**问题：** dong-core 缺少 config 和 db 模块，导致所有依赖它的咚咚项目无法运行

**解决方案：**
- 添加 `dong/config.py` - Config 基类
- 添加 `dong/db/database.py` - Database 基类
- 添加 `dong/db/schema.py` - SchemaManager 基类

**PyPI:** https://pypi.org/project/dong-core/0.2.3/

**经验教训：**
- 发布前必须系统检查所有依赖项
- dong-core 需要包含所有咚咚项目的公共基类
- Config、Database、SchemaManager 是所有咚咚项目的基础

## 关键发现

### dong-core 必须包含的模块

```python
from dong import (
    Config,           # 配置管理基类
    Database,         # 数据库基类
    SchemaManager,    # Schema 管理基类
    DongError,
    ValidationError,
    NotFoundError,
    ConflictError,
    DateUtils,
    json_output,
)
```

### 所有咚咚项目的共同依赖

1. **Config 基类** - 管理用户配置（~/.<cli_name>/config.json）
2. **Database 基类** - SQLite 数据库管理
3. **SchemaManager 基类** - 数据库版本管理
